Side Projects

Hedis

  • Hedis stands for "Hyper Redis". It can retrieve data from ANY database directly via Redis and stores to itself
  • Traditionally, application server retrieves hot data from Redis to reduce unnecessary paths. If Redis doesn't have specific data, application server gets back to retrieve data from original database
  • Hedis can retrieve data from ANY database directly via Redis. Application server retrieves hot data from Hedis server, If Hedis server doesn't have specific data, Hedis retrieves data from original database via ANY connector

Logpush

  • If your program occurs some exception, Logpush will write all your log and send notification to mobile device, it can apply any various log frameworks
  • Combine any various log framework and Google FCM
  • One application server→multiple environments. One environment→multiple subscriptions
